## 1. Краткое описание

Данный проект посвящён численному решению многомерного векторного вязкого уравнения Бюргерса в двух- и трёхмерной постановках. Основной акцент сделан на режиме малых вязкостей, когда конвективная нелинейность начинает доминировать над вязкой диссипацией и в решении формируются области с большими градиентами, близкие к ударным фронтам.

Итоговая реализация находится в ноутбуке:

```text
Burgers_jax.ipynb
```

Код написан на Python с использованием JAX и может запускаться в Google Colab или локально.

## 2. Постановка задачи

В двумерном случае решается векторное вязкое уравнение Бюргерса:

```math
\frac{\partial \mathbf{u}}{\partial t}
+
(\mathbf{u}\cdot\nabla)\mathbf{u}
=
\nu \Delta \mathbf{u},
\qquad
\mathbf{u}=(u,v),
\qquad
\Omega=[0,2\pi]\times[0,2\pi].
```

В координатной форме:

```math
\begin{aligned}
u_t + u u_x + v u_y &= \nu \Delta u,\\
v_t + u v_x + v v_y &= \nu \Delta v.
\end{aligned}
```

В трёхмерном случае рассматривается поле скоростей

```math
\mathbf{u}=(u,v,w),
\qquad
\Omega=[0,2\pi]^3,
```

и система имеет вид:

```math
\begin{aligned}
u_t + u u_x + v u_y + w u_z &= \nu \Delta u,\\
v_t + u v_x + v v_y + w v_z &= \nu \Delta v,\\
w_t + u w_x + v w_y + w w_z &= \nu \Delta w.
\end{aligned}
```

Во всех экспериментах используются периодические граничные условия.

## 3. Используемый численный метод

В проекте реализован спектральный метод Фурье на периодической сетке. Пространственные производные вычисляются в Фурье-пространстве:

```math
\partial_x q =
\mathcal{F}^{-1}\left(i k_x \hat q\right),
\qquad
\partial_y q =
\mathcal{F}^{-1}\left(i k_y \hat q\right),
\qquad
\partial_z q =
\mathcal{F}^{-1}\left(i k_z \hat q\right).
```

Для подавления ошибок алиасинга в нелинейных членах используется правило `2/3 de-aliasing`. Это важно, потому что произведения вида `u * u_x`, `v * u_y` и аналогичные нелинейные члены могут создавать паразитные высокочастотные моды.

Шаг по времени построен по схеме расщепления:

1. половина шага диффузии в Фурье-пространстве;
2. интегрирование нелинейной конвекции методом RK4;
3. вторая половина шага диффузии.

Диффузионная часть считается точно в спектральном пространстве:

```math
\hat q(t+\Delta t)
=
\exp(-\nu |k|^2 \Delta t)\hat q(t).
```

Шаг по времени выбирается по CFL-условию для конвективной части. Для режимов малой вязкости дополнительно используются численные стабилизирующие параметры `eps4` и `filter_strength`. Они отделены от физической вязкости `nu` и отключаются в аналитической валидации.

## 8. Проверка корректности

### 8.1. Аналитическая валидация 2D

Для проверки 2D-солвера используется начальное условие:

```math
u(x,y,0)=\sin x,
\qquad
v(x,y,0)=0.
```

В этом случае решение не зависит от `y`, и двумерная система сводится к одномерному вязкому уравнению Бюргерса:

```math
u_t + u u_x = \nu u_{xx}.
```

Для этой задачи известно аналитическое решение через преобразование Коула — Хопфа:

```math
u(x,t)=-2\nu\frac{\varphi_x(x,t)}{\varphi(x,t)},
\qquad
\varphi_t=\nu\varphi_{xx}.
```

В ноутбуке численное решение сравнивается с аналитическим решением для сеток `64x64`, `128x128` и `256x256`.

Полученные ошибки:

| N | nu | T | steps | err_u_rel_l2 |
|---:|---:|---:|---:|---:|
| 64 | 0.05 | 0.5 | 26 | 4.107087e-06 |
| 128 | 0.05 | 0.5 | 51 | 1.027186e-06 |
| 256 | 0.05 | 0.5 | 101 | 2.592855e-07 |

Ошибка уменьшается при увеличении разрешения, что подтверждает корректность реализации 2D-солвера.

### 8.2. Аналитическая валидация 3D

Для проверки 3D-солвера используется аналогичная 1D-редукция:

```math
u(x,y,z,0)=\sin x,
\qquad
v(x,y,z,0)=0,
\qquad
w(x,y,z,0)=0.
```

В этом случае точное решение также задаётся преобразованием Коула — Хопфа, а компоненты `v` и `w` должны оставаться близкими к нулю.

Полученные ошибки:

| N | nu | T | steps | err_u_rel_l2 | v_rms | w_rms |
|---:|---:|---:|---:|---:|---:|---:|
| 24 | 0.05 | 0.35 | 9 | 5.3e-05 | 0.0 | 0.0 |
| 32 | 0.05 | 0.35 | 12 | 8.0e-06 | 0.0 | 0.0 |
| 48 | 0.05 | 0.35 | 18 | 2.0e-06 | 0.0 | 0.0 |

Результаты показывают уменьшение ошибки при измельчении сетки и сохранение нулевых поперечных компонент.

## 9. Эксперименты при малой вязкости

### 9.1. Основной 2D-эксперимент

В основном 2D-сценарии используется сетка `256x256`, вязкость `nu = 1e-3` и конечное время `T = 0.30`.

Параметры:

```python
nx = 256
ny = 256
nu = 1e-3
t_final = 0.30
cfl = 0.20
eps4 = 2e-6
filter_strength = 18.0
filter_order = 8
```

Визуализируются:

- поле `u`;
- поле `v`;
- модуль градиента;
- центральный профиль;
- история энергии и максимального градиента.

По ходу расчёта энергия остаётся устойчивой, а максимальный градиент увеличивается, что соответствует формированию более резких фронтов при малой вязкости.

### 9.2. Sweep по вязкости

Выполнена серия расчётов при фиксированных численных параметрах и разных значениях физической вязкости:

```python
nu_values = [1e-2, 5e-3, 1e-3]
```

Результаты:

| nu | N | T | steps | energy | max_grad |
|---:|---:|---:|---:|---:|---:|
| 0.010 | 192 | 0.25 | 63 | 0.587999 | 2.326291 |
| 0.005 | 192 | 0.25 | 63 | 0.589621 | 2.337292 |
| 0.001 | 192 | 0.25 | 63 | 0.590923 | 2.346217 |

При уменьшении вязкости максимальный градиент возрастает, что согласуется с физической интерпретацией: меньшая вязкость слабее сглаживает решение.

### 9.3. Основной 3D-эксперимент

В 3D-примере используется сетка `48x48x48`, вязкость `nu = 1e-3` и конечное время `T = 0.15`.

Параметры:

```python
nx = 48
ny = 48
nz = 48
nu = 1e-3
t_final = 0.15
cfl = 0.15
eps4 = 1e-6
```

Для 3D-решения строятся срезы по средней плоскости `z = L_z/2`, центральный профиль, а также графики энергии и максимального градиента.

Выполненные эксперименты показывают, что:

## 11. Ограничения реализации

Данная реализация не является универсальным shock-capturing finite-volume solver. Она основана на спектральном методе Фурье, поэтому естественно подходит для периодических задач и гладких или достаточно регуляризированных решений.

Для очень малых значений вязкости могут потребоваться:

- более мелкая сетка;
- меньший CFL;
- более аккуратный подбор `eps4` и `filter_strength`;
- сравнение с более специализированными методами для задач с разрывами.

3D-расчёт ограничен по размеру сетки из-за более высоких требований к памяти и времени выполнения.
